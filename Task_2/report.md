
## Конфигурация
- Диапазон размеров: 500..10000 с шагом 100
- Диапазон значений: [0, 6000]
- Типы массивов: случайные, обратные, почти отсортированные
- Число прогонов: 5
- Пороговые значения для гибридной сортировки: 5, 10, 20, 30, 50

## Методы
1. `ArrayGenerator` генерирует массивы в трёх конфигурациях.
2. `SortTester` измеряет среднее время выполнения (микросекунды).
3. `mergeSort` - стандартная реализация сортировки слиянием.
4. `hybridMergeSort` - гибридная реализация с переключением на insertion sort.
5. Визуализация выполняется скриптом `visualize.py` с использованием matplotlib/pandas.

## Графики
Графики находятся в каталоге `task1/plots`. Каждый рисунок содержит линии для merge sort и гибридной сортировки с различными порогами.

## Наблюдения
- На случайных массивах гибрид с порогом 20 показывает лучшие результаты до размера ~7000. Для более крупных массивов стандартный merge sort догоняет.
- На обратных массивах гибридные варианты с большими порогами (30, 50) показывают заметный выигрыш.
- На почти отсортированных массивах гибридная сортировка с небольшим порогом (5, 10) быстрее всех.

## Выводы
Пороговое значение 20-30 элементов выглядит оптимальным компромиссом.
